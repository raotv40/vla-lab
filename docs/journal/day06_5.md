## Theory
- **Box Spaces**: Continuous spaces defined by boundary limits (`low` and `high` arrays), shapes, and precision data types.
- **Wrap-around Discontinuity**: Representing angles directly as a scalar $\theta \in [0, 360^\circ]$ creates a boundary leap. A small motion from $359.9^\circ$ to $0.1^\circ$ creates a delta of $-359.8^\circ$, generating destabilizing gradients.
- **Trigonometric Encoding**: Representing $\theta$ as $(\cos(\theta), \sin(\theta))$ keeps state representation continuous across boundaries.

## Observations
- **Data Type Differences**: Observations are double precision (`float64`) because the physics engine integrates state variables with high accuracy. Actions are single precision (`float32`) to match GPU network matrices.
- **Continuous Solution**: Projecting angles to the unit circle $(\cos(\theta), \sin(\theta))$ makes transitions continuous, stabilizing gradients.

---

## Issues Encountered
- **Issue**: Casting float64 observations to float32 is required before neural network forward passes.
- **Solution**: Modern libraries like Stable Baselines3 handle this type casting automatically.
